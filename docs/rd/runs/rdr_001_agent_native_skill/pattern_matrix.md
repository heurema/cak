# RDR-001 Pattern Matrix

Status: exploratory. Patterns are not architecture decisions.

| Pattern | Source(s) | What to copy | What not to copy | Agent-native adaptation | Risk | Informs RDR questions | Evidence status |
|---|---|---|---|---|---|---|---|
| executable external skill library | `src_voyager_arxiv` | Keep reusable behavior outside model weights as interpretable artifacts. | Do not assume stable Minecraft-style APIs or feedback. | Require trace/replay, permissions, and verifier admission before runtime use. | Executable skills can encode brittle or unsafe actions. | RDR-001, RDR-004, RDR-008 | Supported by inspected abstract; needs full-paper and repo review. |
| self-verification before skill admission | `src_voyager_arxiv` | Admission should require execution-grounded feedback. | Do not treat self-verification as independent proof. | Combine self-verification with replay, external verifiers, and human/security review. | Self-confirmation and sampled-test overfitting. | RDR-001, RDR-005, RDR-011 | Supported by inspected abstract; limited detail. |
| workflow induction from trajectories | `src_awm_arxiv` | Mine reusable routines from previous trajectories. | Do not assume retrieved routines are valid in all similar tasks. | Attach preconditions, postconditions, and repair exits to workflow stages. | Wrong-state activation. | RDR-001, RDR-004, RDR-006, RDR-007 | Supported by inspected abstract. |
| workflow brittleness in dynamic UI | `src_hmt_arxiv`; AWM full paper remains a lead | Treat flat memory and site-specific action traces as suspect. | Do not use flat trajectory snippets as universal workflow skills. | Split intent, stage, action grounding, and verifier conditions. | Workflow mismatch under unseen websites or partial observations. | RDR-006, RDR-007 | Supported by HMT abstract; AWM limitation still needs full-paper inspection. |
| Program Function `should_activate` / `intervene` | `src_hasp_arxiv` | Separate activation decision from intervention action. | Do not collapse all skills into hard overrides. | Model active skills as state-conditioned runtime-control objects with audit records. | False-positive activation and overblocking. | RDR-001, RDR-002, RDR-012 | Supported by inspected abstract; exact interface needs full-paper review. |
| action override | `src_hasp_arxiv` | Study hard interventions that modify the next action. | Do not permit overrides without permission, risk, and rollback semantics. | Attach override power to typed effects and policy/verifier gates. | Hidden authority widening. | RDR-002, RDR-012, RDR-017 | Supported by inspected abstract; security evidence missing. |
| context injection | `src_hasp_arxiv` | Study softer interventions that inject corrective context. | Do not assume injected advice will be followed or applied correctly. | Record injected context and measure whether it changed outcome. | Advice can still be ignored or over-applied. | RDR-002, RDR-019 | Supported by inspected abstract. |
| strict filtering / library pollution prevention | `src_hasp_arxiv`, `src_psn_arxiv`, `src_anthropic_agent_skills_docs` | Require validation, teacher/security review, maturity gates, and rollback. | Do not equate "more skills" with progress. | Add quarantine, utility thresholds, counterexamples, versioning, and retirement. | Skill library pollution and stale harmful behavior. | RDR-005, RDR-014, RDR-015 | Partially supported; needs dedicated pollution/security sources. |
| semantic skill contracts | `src_vaso_arxiv` | Represent obligations as explicit external artifacts. | Do not assume all agent skills can be fully formalized. | Treat ContractSpec as one facet of a skill, not the universal skill unit. | Formal interface may require unavailable state observability. | RDR-001, RDR-003, RDR-011 | Supported by inspected abstract; full method not inspected. |
| model-checker filtering | `src_vaso_arxiv` | Filter inconsistent contracts and verify plans against temporal specs when possible. | Do not make model checking a universal prerequisite for all skills. | Use verifier tiers: static checks, replay checks, and formal checks when available. | Overblocking useful skills outside formal domains. | RDR-003, RDR-011, RDR-016 | Supported by inspected abstract. |
| counterexample-driven contract revision | `src_vaso_arxiv` | Convert verifier counterexamples into updates to external skill/contract artifacts. | Do not auto-promote revisions from one failing trace. | Route counterexamples through quarantine, replay, and review before admission. | Overfitting to narrow counterexamples. | RDR-004, RDR-005, RDR-016 | Supported by inspected abstract. |
| maturity-aware update gating | `src_psn_arxiv` | Track skill maturity and gate updates by reliability. | Do not let mature skills become unchangeable when evidence shifts. | Add skill lifecycle states and explicit downgrade/retirement rules. | Stale skills and lifecycle bureaucracy. | RDR-005, RDR-014, RDR-019 | Supported by inspected abstract. |
| rollback validation | `src_psn_arxiv` | Validate that refactors/updates can roll back safely. | Do not treat rollback as only a git concern. | Require rollback plans for active runtime skill versions. | Rollback may fail if skill effects changed external state. | RDR-014, RDR-015, RDR-017 | Supported by inspected abstract. |
| precondition/postcondition memory | `src_hmt_arxiv`, `src_vaso_arxiv` | Attach observable conditions to stages or contracts. | Do not treat natural-language preconditions as checked truth. | Compile conditions into typed-state predicates and verifier observations where possible. | Preconditions may be incomplete or unobservable. | RDR-003, RDR-006, RDR-007, RDR-016 | Supported by inspected abstracts. |
| horizon scanning | `lead_last30days_skill` | Discover recent sources and practitioner signals systematically. | Do not convert discovery into evidence. | Add a Scout role that emits leads only; Archivist must inspect before claim support. | Trend chasing and social-signal bias. | Research process | Seed only; not inspected. |
| multi-review / actor-critic debate | `lead_nitpicker`, `docs/rd/debate_protocol.md` | Use independent reviewers and structured disagreement. | Do not let debate replace source evidence. | Require debate outputs to update ledger, matrices, or unknowns. | Debate theater without evidence. | Research process, RDR readiness | Local protocol inspected; external tool seed not inspected. |
| source ledger / evidence audit | `docs/rd/research_run_protocol.md`, `docs/rd/source_ledger_template.yaml` | Keep claims tied to inspected sources. | Do not cite uninspected seed refs. | Make source ledger the only claim-support artifact. | Bureaucracy if not used to reject weak claims. | Research process, RDR readiness | Supported by local CAK R&D protocol. |
| skill supply-chain analysis | `src_anthropic_agent_skills_docs`; broader security analyses are leads | Treat skills as installable software with trust and audit concerns. | Do not assume "instructions only" means safe. | Require provenance, permission declarations, sandbox boundaries, quarantine, and rollback. | Prompt injection, data exfiltration, malicious code, abandoned dependencies. | RDR-005, RDR-009, RDR-015 | Supported by official docs for baseline risk; broader analyses missing. |
