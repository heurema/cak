# RDR-001 Hypothesis Matrix

Status: exploratory. No hypothesis is adopted.

| Hypothesis | Best supporting patterns | Weaknesses | Required runtime support | Required verifier support | Failure modes | Minimal experiment | Kill criteria |
|---|---|---|---|---|---|---|---|
| H1: Skill = reusable executable behavior. | Voyager executable external skill library; PSN executable symbolic programs. | Does not define activation, permission, admission, or lifecycle by itself. | File/code execution environment, sandboxing, dependency handling, artifact registry, trace capture. | Test execution, replay, static checks, permission/effect checks. | Runs in invalid state; unsafe code; brittle API assumptions; hidden side effects. | Compare executable scripts against text-only and workflow skills on identical failure traces. | Reject as universal unit if executable behavior improves task success but causes high false activation, security risk, or poor transfer. |
| H2: Skill = state-conditioned runtime intervention. | HASP Program Functions; HMT stage-aware precondition validation. | Active intervention can overblock valid behavior and can hide authority widening. | Agent-loop hooks, state observation, activation predicates, intervention modes, audit events, budgets. | Activation precision/recall checks, overblocking metrics, intervention outcome replay. | False-positive intervention; oscillating repair; action override bypasses approval; context injection ignored. | Shadow-mode PFs on recorded traces: compare predicted intervention against actual failures without applying it. | Reject if activation precision is low or false-positive intervention blocks correct agent behavior above threshold. |
| H3: Skill = proof-carrying typed transition. | VASO semantic contracts and model-checker filtering; HMT pre/postconditions; CAK ContractSpec spike as hypothesis. | Many agent environments lack formal state observability or complete propositions. | Typed state, transition model, hook for verifier-gated authority upgrades/downgrades. | Contract validation, model checking where possible, replayed pre/postcondition checks, counterexample handling. | False proof due to missing state; over-formalization; verifier gaps; useful skills blocked because proof unavailable. | Express one invariant as PolicySpec, SkillSpec, ContractSpec, and VerifierPlan; compare clarity and replay behavior. | Reject as universal unit if most useful skills cannot expose meaningful typed transition evidence. |
| H4: Skill = hybrid package containing behavior + contract + verifier + tests + telemetry. | Anthropic Agent Skills package mechanics; HASP runtime intervention; VASO verifier contracts; PSN lifecycle controls. | Broad package may become a vague bundle that hides which part actually works. | Package loader, artifact registry, runtime adapter, permission model, telemetry collector, versioning. | Schema/package validation, script tests, replay tests, contract checks, utility thresholds, security audit. | YAML/package ideology; package accepted but runtime hooks absent; too much authoring burden; stale tests. | Build no runtime: design a paper package for one trace with all facets, then try to map each facet to existing CAK concepts. | Reject if package fields cannot be tied to concrete runtime/verifier semantics or if authoring burden is unbounded. |
| H5: Skill = causal intervention over state-action-outcome patterns. | HASP intervention records; VASO counterexample feedback; HMT workflow mismatch diagnosis. | Causality is hard to infer from traces; correlation may masquerade as cause. | Trace segmentation, state/action/outcome model, intervention logging, counterfactual replay or shadow mode. | Counterexample tests, A/B or shadow evaluation, causal utility estimates, prevented-failure measurement. | Spurious causal claims; recent-trace overfitting; harmful intervention learned from noisy data. | Take repeated failure traces, derive candidate causal interventions, run shadow-mode predictions before admission. | Reject if causal labels cannot predict prevented failures better than simpler typed-state retrieval. |
| H6: Skill = evolving ecological object with lifecycle, maturity, conflict, and retirement. | PSN maturity-aware gating and rollback; HASP stable library evolution; Agent Skills security warnings. | Lifecycle can become process overhead without improving behavior. | Skill registry, lifecycle states, quarantine, versioning, conflict detection, retirement rules, rollback metadata. | Utility estimation, conflict tests, security review, rollback validation, pollution metrics. | Skill library pollution; stale mature skills; conflicting skills; excessive review burden. | Simulate a growing skill library with broad/narrow/conflicting candidates and measure pollution, conflict, utility, and retirement. | Reject if lifecycle machinery cannot reduce pollution or review burden versus a simpler allowlist/quarantine process. |
